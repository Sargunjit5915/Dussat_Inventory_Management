# Firestore Database Schema
## Inventory Management System — Phase 1

---

## Collection: `users`
**Document ID:** Firebase Auth UID (e.g. `abc123xyz`)

| Field          | Type        | Values / Notes                                      |
|----------------|-------------|-----------------------------------------------------|
| `uid`          | string      | Same as document ID                                 |
| `email`        | string      | User's email address                                |
| `displayName`  | string      | Full name entered at registration                   |
| `role`         | string      | `"user"` \| `"admin"`                               |
| `isActive`     | boolean     | `true` — Admin can set to `false` to deactivate     |
| `createdAt`    | timestamp   | serverTimestamp() at registration                   |

**How Admin is set:**
- On registration, if the user provides the correct `VITE_ADMIN_SECRET_KEY`, `role` is written as `"admin"`.
- Otherwise `role` defaults to `"user"`.
- Admins can also manually edit a user document in the Firebase Console to change role.

## Collection: `inventory`
**Document ID:** Auto-generated by Firestore

| Field              | Type      | Values / Notes                                                   |
|--------------------|-----------|------------------------------------------------------------------|
| `name`             | string    | Display name of the item                                         |
| `nameLower`        | string    | `name.toLowerCase()` — used for prefix search queries            |
| `type`             | string    | Category: Electronics, Furniture, Office Supplies, etc.          |
| `quantity`         | number    | Integer ≥ 1                                                      |
| `storageLocation`  | string    | e.g. "Cupboard A3" or "Rack 12"                                  |
| `vendor`           | string    | Where the item was purchased (nullable)                          |
| `companyBrand`     | string    | Manufacturer / brand (nullable)                                  |
| `dateOfAcquisition`| string    | ISO date string: `"YYYY-MM-DD"`                                  |
| `status`           | string    | `"active"` \| `"faulty"`                                         |
| `faultyCategory`   | string    | `null` \| `"returnable"` \| `"fixable"` \| `"ber"`               |
| `addedBy`          | string    | UID of the user who created this document                        |
| `markedFaultyBy`   | string    | UID of user who marked it faulty (null until marked)             |
| `createdAt`        | timestamp | serverTimestamp()                                                |
| `updatedAt`        | timestamp | serverTimestamp() — updated on any field change                  |

**Indexes required (create in Firebase Console → Firestore → Indexes):**
```
Collection: inventory
Fields:     nameLower ASC, __name__ ASC
Query scope: Collection
```
This powers the search-by-name prefix query in Page B.

## Collection: `orderRequests`
**Document ID:** Auto-generated by Firestore

| Field            | Type      | Values / Notes                                                    |
|------------------|-----------|-------------------------------------------------------------------|
| `itemName`       | string    | Name of the requested item                                        |
| `quantity`       | number    | Integer ≥ 1                                                       |
| `priority`       | string    | `"critical"` \| `"high"` \| `"medium"` \| `"low"`                |
| `suggestedVendor`| string    | Vendor suggestion (nullable)                                      |
| `orderLink`      | string    | URL to product page (nullable)                                    |
| `projectName`    | string    | Project name or `"Generic"` if not project-specific               |
| `notes`          | string    | Additional context (nullable)                                     |
| `requestedBy`    | string    | UID of the user who submitted the request                         |
| `status`         | string    | `"pending"` \| `"approved"` \| `"rejected"` \| `"ordered"`       |
| `adminNotes`     | string    | Admin review comments — null until Admin acts (Phase 2)           |
| `createdAt`      | timestamp | serverTimestamp()                                                 |
| `updatedAt`      | timestamp | serverTimestamp()                                                 |

---

## Security Rules (see `firestore.rules`)
- **`users`**: A user can read/write only their own document. Admins can read all.
- **`inventory`**: Any authenticated user can read and create. Only the creator or an Admin can update/delete.
- **`orderRequests`**: Any authenticated user can create and read their own. Admins can read all and update status.
